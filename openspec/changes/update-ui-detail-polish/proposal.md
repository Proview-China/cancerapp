## Why
前端界面整体风格已定型（火山口+池塘），但在交互细节上仍有可感知的瑕疵：侧边栏可被拖到过窄影响可读性，卡片/列表的间距与排版不够统一，文字溢出与多行截断缺少规则，影响信息密度与观感。

## What Changes
- 侧边栏：设置一致的最小宽度（与逻辑约束一致），网格/容器在 CSS 层也做下限保护；拖拽时严格 clamp。
- 卡片与列表：统一内边距/行高/字号；为标题与摘要增加截断（line-clamp）；对齐操作区域；微调 hover/active 阴影以契合池塘风格。
- DevTools 检查：使用 Chrome DevTools MCP 校验布局边界（最小宽度、溢出截断、可点击区域），输出一次可复现的检查步骤。

## Impact
- 影响文件：`frontend/src/components/AnalysisSidebar.{tsx,css}`, `frontend/src/App.tsx`（如需常量同步），以及与卡片/列表相关的 CSS。
- 用户可见：侧边栏在任何情况下不会低于最小阅读宽度；卡片/列表文字更规整、操作区更稳定；视觉对齐更统一。
